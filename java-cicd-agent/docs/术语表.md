# 术语表

本术语表用于统一 `java-cicd-agent` 文档中的常见概念与表达方式，避免同一个意思在不同文件里换着叫，最后把人看成腌制 CI 鱼干。

## 发布（Release）

指从“已有源码版本”出发，完成：
- 构建
- 镜像生成
- 镜像推送
- 部署推进
- 验证结果

在本文档里，“发布”是比“部署”更完整的概念。

---

## 部署（Deploy）

指把某个镜像版本真正应用到 Kubernetes 或 GitOps 目标环境中。

例如：
- `kubectl apply`
- 更新 GitOps 仓库中的镜像标签
- 等待 Argo CD 同步

简单说：
- 发布 = 从源码到结果的完整链路
- 部署 = 链路中的落地动作

---

## 构建（Build）

指把源码编译成可运行制品的过程。

在 Java 项目里通常是：
- `mvn package`
- 产出 JAR

---

## 镜像构建（Image Build）

指基于 Dockerfile 或 Kaniko，将应用制品打包成容器镜像。

---

## 镜像推送（Push Image）

指把构建好的镜像上传到镜像仓库，例如 Harbor。

---

## 镜像仓库（Registry）

用于存放容器镜像的服务。

在本项目中，默认指：
- Harbor

---

## GitOps

一种以 Git 仓库作为“环境期望状态来源”的部署方式。

在本文档语境里：
- 源码仓库存放应用代码
- GitOps 仓库存放部署清单
- Argo CD 监听 GitOps 仓库并同步到集群

---

## GitOps 仓库

专门存放部署 YAML、kustomization、Argo CD Application 等内容的仓库。

不用于存放 Java 业务源码。

---

## Argo CD Application

Argo CD 中定义的一个应用对象，用于描述：
- 从哪个 Git 仓库读
- 读哪个分支
- 读哪个路径
- 同步到哪个集群/命名空间

---

## 同步（Sync）

指 Argo CD 将 GitOps 仓库中的目标状态应用到 Kubernetes 集群的过程。

常见状态包括：
- `Synced`
- `OutOfSync`

---

## 健康（Healthy）

指 Argo CD 或应用层面认为当前部署状态健康。

注意：
- `Synced` 不一定代表真正可用
- 最终仍应结合 rollout 与健康检查接口判断

---

## Rollout

指 Deployment 从旧版本滚动到新版本的过程。

常见检查方式：
- `kubectl rollout status`

---

## 健康检查（Health Check）

指通过应用接口确认服务是否真正可用的检查。

常见路径：
- `/actuator/health`
- `/health`
- `/healthz`

在本文档中，统一使用“健康检查”这一中文表述。

---

## 初始化（Bootstrap）

指某个系统或仓库的首次准备动作。

例如：
- 初始化 GitOps 仓库
- 初始化 Harbor
- 初始化 Argo CD Application

在本文档中，统一翻译为“初始化”。

---

## 预检（Preflight）

指正式安装或发布前的前置检查。

例如：
- 检查磁盘
- 检查网络
- 检查 TLS / DNS / 存储规划

---

## Robot Account

Harbor 中用于 CI 自动推镜像的机器账户。

在本文档中，保留英文术语 `robot account`，必要时可补充说明为“机器账户”。

---

## Pipeline

指 Jenkins 或 CI 系统中的自动化流程。

在本文档中，一般翻译为：
- 流水线
- Pipeline（在具体配置场景中保留）

---

## Agent

指执行某类自动化任务的代理角色。

例如：
- Java 代码 Agent
- Java CI/CD Agent

在本文档中保留 `Agent` 这个词，不强行替换成“代理程序”，避免读起来像上世纪翻译教材。
