# 架构图

## 短版架构图

```text
[上游 Java 代码 Agent]
        │
        │ 产出代码 / git ref
        ▼
[demo-java-service 源码仓库]
        │
        │ release-request.json
        ▼
[Java CI/CD Agent]
        ├─ 拉源码
        ├─ Maven 构建
        ├─ Docker 构建
        ├─ Push Harbor
        ├─ 更新 GitOps 仓库 / 或直接部署
        └─ 验证 rollout + health
                │
                ├────► [Harbor]
                ├────► [demo-java-service-gitops]
                │              │
                │              ▼
                │         [Argo CD]
                │              ▼
                └──────────► [Kubernetes]
```

---

## 角色说明

### 上游 Java 代码 Agent
负责：
- 编写和修改 Java 业务代码
- 提交源码变更
- 产出可发布的 git ref

不负责：
- 构建镜像
- 推送 Harbor
- 更新 GitOps 仓库
- 部署 Kubernetes

### Java CI/CD Agent
负责：
- 接收发布请求
- 准备源码
- Maven 构建
- Docker 构建与推送
- 直发部署或更新 GitOps 仓库
- 检查 Argo CD / rollout / health
- 输出发布结果

---

## 仓库职责

### demo-java-service
应用源码仓库，用于：
- 存放 Java 源代码
- 存放 pom.xml、Dockerfile
- 作为构建输入

### demo-java-service-gitops
部署仓库，用于：
- 存放 Kubernetes YAML
- 声明目标环境应使用的镜像版本
- 供 Argo CD 监听与同步
