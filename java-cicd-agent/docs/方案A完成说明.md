# 方案 A 完成说明

本文件用于记录本轮“术语统一扫尾”的处理范围与统一口径。

## 本轮重点统一的术语

- 发布：指从源码版本到最终结果的完整链路
- 部署：指把镜像或配置真正落到目标环境的动作
- 健康检查：统一作为 health check / healthcheck 的中文表述
- 初始化：统一作为 bootstrap 的中文表述
- 同步：作为 Argo CD sync 的中文表述，状态值 `Synced` 保留原文
- 滚动发布状态：作为 rollout 的中文说明性表述，命令 `kubectl rollout status` 保留原文
- 镜像仓库：作为 registry 的中文表述，Harbor 仍保留英文产品名

## 统一原则

1. 面向人阅读的说明文档优先使用中文术语。
2. 面向脚本、配置和状态值的关键技术锚点保留英文原文。
3. 避免把“发布”和“部署”混用成一个意思。
4. 对 `Synced`、`Healthy`、`Application`、`Deployment` 等对象名或状态名，保留英文原文，并在上下文中补中文说明。

## 当前结果

- 文档整体已经完成全中文化。
- 主干高频术语已经做过一轮统一。
- 文档中仍会保留必要的英文技术锚点，以保证跨 Agent / 跨模型理解稳定性。
