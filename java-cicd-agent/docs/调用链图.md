# 调用链图

## GitOps 模式

```text
release-request.json
  ↓
run_release_from_request.sh
  ↓
prepare_source.sh
  ↓
release.sh
  ├─ build_java.sh
  ├─ build_image.sh
  ├─ push_image.sh
  ├─ update_gitops_image.sh
  ├─ check_argocd_app.sh
  └─ verify_deploy.sh
```

---

## Direct 模式

```text
release-request.json

run_release_from_request.sh

```

---

## 脚本职责简表

- `run_release_from_request.sh`：读取请求文件并驱动整个发布流程
- `prepare_source.sh`：拉取源码仓库并切到指定 git ref
- `release.sh`：发布总入口，统一编排各阶段
- `build_java.sh`：执行 Maven 构建
- `build_image.sh`：执行 Docker 镜像构建
- `push_image.sh`：推送镜像到 Harbor
- `deploy_k8s.sh`：直发模式下执行 Kubernetes 部署
- `update_gitops_image.sh`：GitOps 模式下更新镜像标签并提交仓库
- `check_argocd_app.sh`：等待 Argo CD 应用同步并检查状态
- `verify_deploy.sh`：检查 rollout 与健康状态
- `rollback.sh`：失败时按策略执行回滚
