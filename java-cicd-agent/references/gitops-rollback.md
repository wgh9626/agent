# GitOps 回滚

## 原则

在 GitOps 部署模型中，回滚应由 Git 状态驱动，而不是通过在集群里临时手改配置完成。

## 推荐的回滚方式

### 方式一：回退发布提交
- 回退 GitOps 仓库中修改镜像标签的那次提交。
- 推送这次 revert 提交。
- 由 Argo CD 将状态收敛回上一个镜像版本。

### 方式二：重新提交上一个已知正常镜像
- 将目标 Deployment 清单中的镜像改回最后一个已知正常版本。
- 提交并推送该变更。
- 由 Argo CD 执行声明式回滚。

## 默认应避免的做法

- 直接手工 `kubectl edit`
- 在 Git 之外直接 patch 镜像
- 混用多种回滚方式，导致 Git 与集群状态不一致

## 回滚后的验证

- Argo CD 应用状态变为 `Synced`（同步成功）
- Argo CD 应用状态变为 `Healthy`（状态健康）
- 服务健康检查接口返回成功
- 在发布记录或事故记录中说明回滚原因
