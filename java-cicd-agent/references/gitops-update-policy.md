# GitOps 更新策略

## 原则

CI 在一次发布中，只应更新这次发布所需的镜像引用。
不应顺手修改 GitOps 仓库中无关的运行时配置。

## 允许的变更

- 将 `__IMAGE__` 替换为明确的镜像标签
- 只更新目标服务的单个 Deployment 清单
- 生成一条清晰的发布提交信息

## 默认不允许的变更

- 自动修改副本数
- 自动修改资源限制
- 自动修改命名空间
- 在未明确要求的情况下，一次发布更新多个服务

## 推荐的提交信息

`release(<service-name>): update image to <tag>`

## 回滚原则

优先使用 Git revert，或重新提交一条指向上一个镜像标签的新提交。
然后由 Argo CD 以声明式方式收敛回滚。
