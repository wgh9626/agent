# k3s 资源规模建议

## 小型首版平台

适用于单节点平台，同时运行：
- k3s
- Harbor
- Jenkins
- Argo CD
- 一个或多个小型测试服务

建议起步配置：
- 8 vCPU
- 16 GB RAM
- 200 GB SSD

## 为什么建议这个规模

- Harbor 需要存储和一定内存余量。
- Jenkins 构建任务会带来 CPU 和内存峰值。
- Argo CD 本身较轻，但也会持续占用基础资源。
- Java 服务需要为 Pod、探针和 rollout 重叠留出空间。

## 更小的实验配置

如果只是做实验，也可以从更小的规格开始，例如：
- 4 vCPU
- 8 GB RAM
- 100 GB SSD

但在镜像构建和并发活动场景下，会更容易触碰资源上限。

## 持久化要求

至少需要为以下内容做持久化：
- Harbor 存储
- Jenkins Home

## 建议

如果预算允许，不要把第一版平台卡得太死。
资源被饿瘦的 CI/CD 平台，就像一个灶台上挤了八个厨师，最后谁都做不好饭。
