# 多环境扩展

## 第一原则

先只做 `test`。
只有当第一条发布路径稳定后，再增加更多环境。

## 建议的扩展顺序

1. `test`
2. `staging`
3. `prod`

## GitOps 结构选项

```text
<service>-gitops/
└── apps/
    ├── test/
    ├── staging/
    └── prod/
```

## 各环境建议差异

### test
- 迭代最快
- 可允许自动同步
- 对规模要求相对宽松

### staging
- 更接近生产配置
- 可要求人工晋级
- 验证规则更严格

### prod
- 审批流程更严格
- 不允许在 Git 之外自动修改配置
- 通过 GitOps 提交历史处理回滚

## 规则

在 `test` 环境发布路径还不稳定之前，不要过早引入多环境复杂度。
